# TRACE evaluation — provenance & how to reproduce

This directory holds **one** canonical evaluation, not a pile of conflicting
reports. Everything here is regenerated by a single command.

## Canonical artifacts

| File | What it is |
|------|------------|
| `manifest.json` | The labeled holdout (63 images: 13 helmet + 50 plate) |
| `seatbelt_manifest.json` | Cropped seatbelt-classifier holdout (37 images) |
| `results.json` | Machine-readable detection/violation/OCR results |
| `REPORT.txt` | Human-readable canonical report |
| `eval-summary.json` | Dashboard summary (Settings → Performance) |
| `REPORT-seatbelt.txt` / `seatbelt_results.json` | Seatbelt-only eval |

> The 11 earlier `REPORT-*.txt` / `results-*.json` variants were removed.
> They were produced under different model/API states and an old metric, so
> their numbers disagreed. Git history preserves them if ever needed.

## Regenerate

```bash
pip install -r requirements.txt -r requirements-ml.txt
pip install -e .
export ROBOFLOW_API_KEY=...                     # only for config/roboflow.yaml
python scripts/run_full_eval.py --config config/roboflow.yaml
python scripts/eval_seatbelt.py  --config config/viovision.yaml
```

`run_full_eval.py` writes `results.json`, `REPORT.txt`, and `eval-summary.json`
together, so they can never drift apart.

## Building a credible full eval set (the next data task)

To report honest Accuracy / Precision / Recall / F1 / mAP across all seven
violations, extend `manifest.json` with **balanced, fully-labeled** samples.

Sample schema (per image):

```jsonc
{
  "id": "junction12_0093",
  "image": "images/junction12_0093.jpg",
  "width": 1920, "height": 1080,
  "eval_kind": "scene",                 // scene | helmet_violation | plate_detection
  "detections_gt": [                    // EVERY vehicle/person box, not just one
    {"cls": "car",        "bbox": [x1,y1,x2,y2]},
    {"cls": "motorcycle", "bbox": [x1,y1,x2,y2]},
    {"cls": "person",     "bbox": [x1,y1,x2,y2]}
  ],
  "violations": ["red_light", "no_helmet"],   // all true violations in frame
  "detail": {"plate_text": "MH12AB1234"}      // when a plate is legible
}
```

Targets for a defensible submission:
- **Detection:** ≥200 fully-labeled frames spanning day/night/rain/blur.
- **Violations:** ≥30 positive + 30 negative examples **per type** (7 types).
  Temporal types (wrong-side, parking, stop-line, red-light) need short clips
  or frame sequences plus per-camera calibration in `config/*.yaml`
  (`stop_line`, `lane`, `signal`, `no_parking_zones`).
- **OCR:** the 41 `plate_text` samples already present; add more per state code.

`scripts/run_full_eval.py` consumes this schema as-is — no code changes needed,
the numbers populate automatically once the labels exist.
